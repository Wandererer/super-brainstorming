# super-brainstorming

Claude Code에서 돌아가는 멀티에이전트 브레인스토밍 플러그인. 주제를 던지면 여러 에이전트가 병렬로 아이디어를 발굴·발산하고, 코드 게이트가 차갑게 수렴시켜 점수 랭킹과 컨셉 브리프로 돌려줍니다.

**발산은 뜨겁게, 수렴은 차갑게. 근거는 URL로.**

## 설치

```
/plugin marketplace add Wandererer/super-brainstorming
/plugin install super-brainstorming
```

설치 후 Claude Code를 재시작하면 `/super-brainstorming`이 활성화됩니다.

### 첫 실행

플러그인을 처음 쓸 때, 이 저장소에 GitHub ⭐를 눌러줄지 **한 번만** 묻습니다 — 전적으로 선택이며, 별과 무관하게 모든 기능은 동일하게 동작합니다. 명시적으로 예라고 답한 경우에만, 로컬 [GitHub CLI](https://cli.github.com/)(`gh`) 자격 증명으로 `gh api`를 호출해 저장소에 별을 답니다. `gh`가 설치돼 있지 않거나 로그인돼 있지 않으면, 선택만 기록되고 아무것도 전송되지 않습니다 — 별 달기만 건너뜁니다. 답변은 `~/.super-brainstorming-setup/`에 저장돼, 다시 묻지 않습니다.

## 두 가지 발산 모드

### 🔍 검색 발굴 — `research_first`

"여러 곳을 뒤져서, 근거 있는 아이디어만." 마이너 에이전트 6종이 서로 다른 소스 클래스를 병렬로 팝니다 — 5종은 **수요**(불만·빈틈)를, 1종은 **공급**(이미 존재하는 해법)을.

| 마이너 | 파는 곳 | 캐는 것 |
|--------|---------|---------|
| 리뷰 | 앱스토어, G2, 스토어 리뷰 | 1-3점 리뷰, "이것만 됐으면" 패턴 |
| 커뮤니티 | Reddit, HN, 포럼 | 반복 불만, 사용자가 직접 만든 우회로 |
| 경쟁사 빈틈 | 경쟁 제품 5-8개 | 전부가 안 하는 것과 그 이유 |
| 트렌드 | 최근 1-2년의 변화 | 왜 지금 가능해졌나 (타이밍 재료) |
| 시장 데이터 | 리포트, 정량 지표 | 성장 카테고리, 가격 벤치마크 |
| **기존 해법** | 앱스토어 차트, Product Hunt, GitHub | 니치별 기존 제품 — 가격·무료 여부·traction (me-too의 천적) |

캐낸 근거는 전부 출처 URL과 함께 `evidence_ledger.jsonl`에 쌓이고 — **URL 없으면 카드 금지** — 합성 렌즈 3종(불만→제품, 빈틈→쐐기, 트렌드×불만)이 그 근거에서만 아이디어를 만듭니다. 근거를 인용하지 않은 아이디어는 게이트가 반려합니다.

```
/super-brainstorming research AI 코딩 도구 시장의 미충족 니즈
```

### 🎨 사고 렌즈 — `creative` (기본)

검색 없이 순수 발산. 13개 렌즈 라이브러리 — 역발상 · 제1원리 · 타업계 이식 · 불만 고고학 · 제약 뒤집기 · 미래 역산 · SCAMPER · 극단 자원 · 인접 가능성 · 안티 페르소나에 와일드카드(무작위 자극 · 빌런 · 10x) — 에서 렌즈 에이전트 4~8개를 뽑아, 각자의 사고법으로 발산합니다. 발산 중 비판은 금지 — red-team은 수렴 단계에만 존재합니다.

```
/super-brainstorming 팀 회고 문화를 바꿀 방법
```

## 아이디어가 통과해야 하는 관문

이 플러그인의 핵심은 발산이 아니라 **수렴을 코드에 맡긴 것**입니다. 모든 아이디어는 다섯 관문을 지나야 쇼트리스트에 오릅니다.

1. **선행기술 대조** — 아이디어 클러스터 단위로 기존 제품을 검색해 대조합니다. 경쟁사도 URL 붙은 카드로 등록해야 하고(지어내기 불가), 각 아이디어에 포화도(0-5)가 매겨집니다. 두 모드 공통 — 발산은 자유롭게, 수렴은 근거로.
2. **Red-team** — 구조적 결함(제약 위반·자기모순·anti-goal 저촉)을 찾아 이유와 함께 기각합니다. "어렵다"는 기각 사유가 아니지만, **성숙한 무료 제품이 이미 잘하는데 쐐기가 없으면** 기각 사유입니다.
3. **독립 심사 2-3명** — 서로의 점수를 못 보는 상태로 5축 채점: impact `.30` · feasibility `.25` · novelty `.20` · fit `.15` · timing `.10`. **novelty는 첨부된 선행기술 목록 기준으로만** — 머릿속 추정 금지.
4. **`validate_ideas.py`** — 심사 중앙값 × 가중치로 랭킹을 **계산**합니다. **포화된 니치(무료/OSS 경쟁자 포함)의 아이디어는 novelty가 코드로 상한 처리되고, 기존 해법 대비 쐐기(wedge) 없이는 통과하지 못합니다.** 쇼트리스트 파일은 이 게이트를 통과해야만 생성되므로, LLM이 임의로 승자를 고르는 일 자체가 불가능합니다.
5. **Wildcard seat** — 상위권이 전부 안전한 아이디어면, 마지막 1석을 가장 과격한(wildness≥4) 아이디어로 코드가 교체합니다.

심화 단계에도 관문이 있습니다. 컨셉당 실행 접근법 2-3개를 비교하고, YAGNI로 MVP를 깎고, 가장 위험한 가정마다 2주 내 검증 방법을 붙입니다. 그리고 **HARD-GATE — 브리프가 승인되기 전에는 어떤 구현도 시작하지 않습니다.** 마감 직전에는 `eval_briefs.py`가 placeholder 잔존·근거 누락·문서 정합성을 채점해, FAIL이면 마감 자체를 막습니다.

## 커맨드

| 커맨드 | |
|--------|---|
| `/super-brainstorming research [주제]` | 검색 발굴 모드로 바로 시작 |
| `/super-brainstorming [주제]` | 주제를 보고 모드를 추천 |
| `/super-brainstorming frame` | 막연한 주제를 HMW 프레임으로 정형화 |
| `/super-brainstorming resume [id]` | 중단한 세션 이어하기 |
| `/super-brainstorming status` | 전체 세션 현황 |

"[주제] 검색해서 아이디어 발굴해줘", "[주제] 브레인스토밍" 같은 자연어로도 트리거됩니다.

## 무엇이 나오나

`BRAINSTORM/{주제}_{시각}/outputs/`:

| 파일 | 내용 |
|------|------|
| `00_executive_summary.md` | 요약 + 쇼트리스트 표 (점수·wildness·근거) |
| `01_idea_catalog.md` | 30+ 아이디어 전체 — 클러스터, 점수, red-team 반론 |
| `02_concept_briefs/` | 선택 컨셉 심화: 접근법 비교, MVP In/Out, 위험 가정 + 2주 검증법 |
| `03_parking_lot.md` | 기각·보류 — 이유와 부활 조건까지 (기각도 기록입니다) |
| `04_evidence_digest.md` | (검색 발굴) 어디를 뒤졌고 무엇이 나왔는지, 전부 URL과 함께 |
| `handoff.md` | 다음 단계용 검증 쿼리 |

세션 상태는 `state.json`에 게이트 서명과 함께 저장돼, 중단해도 `resume`으로 이어집니다.

## 다음 단계로 잇기

브리프가 끝이 아닙니다. 선택한 컨셉마다 가장 위험한 가정을 검증하는 리서치 쿼리가 `handoff.md`에 자동 생성됩니다. 딥리서치 도구가 있다면 그대로 이어 실행할 수 있고, 없다면 각 브리프의 "2주 내 검증 방법"이 가장 싼 실험입니다.
